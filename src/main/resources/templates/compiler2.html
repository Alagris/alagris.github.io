    Quick tour

    Solomonoff is a regular language with many enhancements. It compiles to symbolic weighted transducers. Use can use |, *, ?, + just like in UNIX regexes. Use can even use square brackets with ranges like [a-z] or [0-1] but you are not allowed to combine them together. For instance the following are not valid [a-z0-1], [a], [a-z0]. You also need to mind that all literal strings have to be enclosed in single quotes, like 'a', 'str\ning' or the empty string ''. Hence the UNIX regex (AA[0-9a-z]BB+)* might be equivalently written in Solomonoff as ('AA' ([0-9]|[a-z])'B' 'B'+)*.

    Every regular expression needs to be defined as a variable 
    x = ('AA' ([0-9]|[a-z])'B' 'B'+)*
    You may group your regular expressions into multiple variables. There is no need to separate the definitions with semicolons like in some other languages 
    x = 'he' 'l'+ 'lo'
y = 'world'
hello_world=x ' ' y
    You can click on the above examples to copy them to code editor. Then press "compile" button and you will be able to test them in REPL. 
    :eval hello_world 'hellllllo world'
    Type the above command in console (or click to automatically copy it). Notice that typing 
    :eval x 'hellllllo'
    results in Transducer 'x' not found!. Below the REPL console you may see the current list of defined variables. Whenever you use a variable as a building block for another expression, the variables gets "consumed". In order to prevent this, you can explicitly copy the variable with double exclamation marks 
    x = 'he' 'l'+ 'lo'
y = 'world'
hello_world2= !!x ' ' !!y
    Now if you compile this, you will be able to run 
    :eval x 'hellllllo'
    Be careful not to redefine variables that are already defined. Notice that we created hello_world2 instead of hello_world, because the latter was already created and was not yet consumed. You can only redeclare a variable after it was consumed. Alternatively you may also run
    :unset hello_world
    To see the full list of available REPL commands, you may type 
    :?
    The core feature of Solomonoff are of course the transducers. This means that you can specify output printed by any regular expression. For example 
    z = 'a':'b'
    will print b as the output whenever a is matched. Try it with 
    :eval z 'a'
    Another example might be
    z2 = ('a':'b')*
z3 = [0-9]:'cde'
z4 = :'prefix' !!z2 !!z3
z5 = '':'01234'
    The output get concatenated if specified one after the other
    z6= :'a' :'b' '0'
